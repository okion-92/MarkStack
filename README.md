# MarkStack Desktop

MarkStack Desktop is a Windows desktop Markdown workspace app. It focuses on local Markdown editing, previewing, workspace search, embedded assets, and export workflows.

MarkStack 桌面端是一个 Windows 本地 Markdown 工作区应用，重点覆盖 Markdown 编辑、预览、工作区搜索、图片嵌入和导出流程。

## Current Version

- App version: `0.1.18`
- Package name: `markstack-desktop`
- Target platform: Windows desktop
- Installer format: NSIS `.exe`
- Main development branch: `dev`
- Stable branch: `main`

## Environment

Recommended local environment:

- Windows 10/11
- Node.js `24.11.1` or newer compatible Node 24 runtime
- npm `11.6.2` or newer compatible npm 11 runtime
- PowerShell 7 or Windows PowerShell
- Git

The project is built and verified with Node `24.11.1` and npm `11.6.2` on Windows.

## Features

Document editing:

- Open and save `.md`, `.markdown`, `.mdown`, `.mkd`, and `.txt` files.
- Drag a Markdown/text file into the window to open it as the active document.
- Prompt before replacing unsaved content or closing the current opened file.
- New documents open in split view; opened files default to preview-only view.
- Close the current opened file from the file header.
- Keep a recent-file list and remove missing recent files after failed open attempts.
- Store unsaved drafts in local storage and offer restore after restarting the app.

Editor and formatting:

- CodeMirror Markdown editor with line numbers, folding, active-line highlight, and autocomplete.
- Built-in find/replace through CodeMirror search.
- Formatting toolbar for headings, paragraph text, bold, italic, strikethrough, inline code, links, images, quotes, lists, code blocks, alignment, font size, font family, and text color.
- Text color formatting works for selected text inside inline code spans, such as the `A` inside `` `A` ``.
- Clipboard image paste inserts embedded base64 data URI images.
- Image picker supports `png`, `jpg`, `jpeg`, `gif`, `webp`, `svg`, and `bmp`, with a 10 MB embedded-image limit.

Preview:

- Real-time Markdown preview.
- Split, editor-only, and preview-only modes.
- Synchronized editor/preview scrolling in split mode.
- Heading outline generated from Markdown headings.
- Markdown HTML is sanitized before preview rendering.
- External preview links open in the system browser and cannot navigate the app window.
- Code blocks are highlighted with highlight.js.
- Mermaid diagrams render from fenced code blocks using `mermaid` as the language name.

Workspace:

- Open a local folder as a Markdown workspace.
- Sidebar lists supported Markdown/text files in the workspace.
- Reopen the last workspace from the sidebar after app restart.
- Full-text search across workspace Markdown/text files.
- Workspace search limits are intentionally bounded for responsiveness: up to 1,000 workspace files, 2 MB per searched file, and 100 displayed matches.
- Ignored workspace directories include `.git`, `dist`, `dist-electron`, `node_modules`, and `release`.

Export:

- Export the rendered preview as standalone HTML.
- Export the rendered preview as PDF.
- PDF export supports paper size (`A4` or `Letter`), orientation (`portrait` or `landscape`), and margin mode (`default` or `none`).
- Export uses the sanitized rendered preview HTML. If Mermaid has already rendered in preview, the exported HTML/PDF uses that rendered SVG output after another sanitization pass.
- Exported HTML/PDF files are revealed in Windows File Explorer after export.

Installer:

- Windows NSIS installer generated by electron-builder.
- Assisted installation flow with selectable install directory.
- Upgrade flow detects an existing install and can run the previous uninstaller before installing the new version.
- Generated artifacts are ignored by Git.

## Security Notes

- Electron renderer runs with `contextIsolation: true`, `nodeIntegration: false`, and sandbox enabled.
- The preload layer exposes a small IPC API through `window.markstack` instead of exposing Node APIs to the renderer.
- App-window navigation is blocked for non-app URLs; allowed external protocols are `http:`, `https:`, and `mailto:`.
- Markdown preview HTML is sanitized with DOMPurify before insertion into the page.
- Exported preview HTML is sanitized again before being written or printed.
- The hidden PDF print window disables JavaScript and normalizes PDF options through a main-process whitelist.
- Embedded images are validated by extension and size before being converted to data URI content.
- `npm audit` currently reports `0` vulnerabilities after upgrading Electron, electron-builder, and esbuild.

```bash

```

## Development Workflow

Use `dev` for active development:

```bash
git switch dev
git pull origin dev
```

Before committing:

```bash
npm run typecheck
npm run build
npm audit
```

Commit and push:

```bash
git status
git add .
git commit -m "Describe your change"
git push origin dev
```

Merge release-ready work into `main` after review:

```bash
git switch main
git pull --ff-only origin main
git merge --no-ff dev
git push origin main
git switch dev
```
